# Brain English — PRD (Product Requirements Document)

**작성일**: 2026-07-01
**버전**: v0.1 (MVP)
**작성 목적**: 뇌과학 기반 학습 원리(간격 반복, 능동적 회상, 이해 가능한 입력)를 적용한 아동용 영어 학습 앱의 제품 요구사항 정의

---

## 1. 배경 및 문제 정의

기존 영어 학습 앱은 대부분 정해진 커리큘럼(코스)을 순서대로 따라가는 방식이다. 이 방식의 한계는 다음과 같다.

- **학습 후 방치**: 한 번 배운 표현을 다시 떠올리게 하는 장치가 없어 망각 곡선(Ebbinghaus forgetting curve)대로 잊혀진다.
- **아이의 실제 학습 맥락과 분리**: 아이가 실생활(책, 교재, 간판, 카드 등)에서 마주치는 영어가 아니라 앱이 미리 정해준 콘텐츠만 학습한다.
- **대화형 AI의 태도 문제**: 기존 AI 회화 앱/챗봇은 아이가 머뭇거리면 답을 먼저 말해주거나 대신 문장을 완성해준다. 이는 성인 외국어 학습자에게는 효율적일 수 있으나, **모국어를 처음 배우는 아기를 대하듯 기다려주고 유도해야 하는 아동**에게는 오히려 스스로 말할 기회를 빼앗는 역효과를 낸다.

## 2. 제품 비전

> 아이가 "지금 이 순간" 마주친 영어를 사진 한 장으로 학습 콘텐츠화하고, 뇌과학적으로 검증된 간격 반복(spaced repetition)과 능동적 회상(active recall)을 통해 장기 기억으로 전환시키며, 아이의 언어 발달 속도를 존중하는 AI와 대화하며 실제로 사용해보는 영어 학습 루프를 만든다.

**핵심 차별점 3가지**

1. **캡처 기반 학습**: 정해진 코스가 아니라 아이가 실제로 만난 영어 텍스트(사진 → OCR)를 학습 단위로 삼는다.
2. **간격 반복 리마인드**: 학습 직후가 아니라 며칠 뒤, 정해진 간격으로 "그때 배운 거 기억나?"를 능동적으로 회상시킨다.
3. **아이 눈높이 AI 튜터**: 아이가 머뭇거려도 먼저 답을 주지 않고, 기다리고, 힌트를 주고, 아이의 말을 자연스럽게 확장해주는 — 마치 모국어를 배우는 유아를 대하는 보호자 같은 대화 방식.

## 3. 타겟 사용자

**주 사용자 (아이)**
- 문법 지식은 거의 없음
- 간단한 영어 단어/문장을 어느 정도 읽을 수 있음
- 짧은 문장 수준의 말하기가 가능함
- 스스로 스마트폰/태블릿 앱을 조작할 수 있는 연령/숙련도 (보호자 상시 개입 불필요)
- 한국에 거주하며 한국어를 모국어로 사용

**부 사용자 (보호자)**
- 아이의 기기에 앱을 설치/설정해주는 역할
- MVP에서는 학습 과정에 직접 개입하지 않음 (자녀가 스스로 사용)
- 추후 버전에서 진행 상황 확인 니즈 존재 가능 (범위 외, 4.1 참고)

**시장/언어**: 한국어 UI, 한국 거주 아동 대상. AI와의 대화 및 학습 콘텐츠 자체는 영어.

## 4. 목표 및 성공 지표

| 목표 | 지표 (MVP 단계 참고용) |
|---|---|
| 학습 루프 완주율 | 캡처 → 초기 학습 → 첫 리마인드까지 도달하는 학습 아이템 비율 |

## 5. 학습 과학적 근거 (Learning Science Foundations)

PRD 전반의 기능 설계는 아래 원리에 근거한다. 각 기능 섹션에서 어떤 원리를 구현하는지 명시한다.

1. **간격 반복 효과 (Spaced Repetition Effect)**: 학습 직후보다 일정 간격을 두고 재노출할 때 장기 기억 정착률이 높다 (Ebbinghaus, Cepeda et al.).
2. **능동적 회상/시험 효과 (Retrieval Practice / Testing Effect)**: 답을 다시 보여주는 것보다 스스로 떠올리려 시도하는 것이 기억을 강화한다 (Roediger & Karpicke).
3. **생성 효과 (Generation Effect)**: 수동적으로 정답을 인식하는 것보다 스스로 산출(말하기/쓰기)할 때 기억이 더 강하게 남는다.
4. **바람직한 어려움 (Desirable Difficulty)**: 약간 힘들게 회상해야 학습 효과가 크다. 너무 쉬운 즉시 반복이나 너무 어려운 장기 방치 모두 비효율적.
5. **이해 가능한 입력 (Comprehensible Input, i+1)**: 아이가 이미 아는 것 바로 위 난이도의 입력이 습득을 촉진한다 (Krashen).
6. **아동지향어/모성어 대화 방식 (Child-Directed Speech / Scaffolding)**: 유아의 모국어 습득 과정에서 보호자는 아이의 침묵을 기다리고(wait time), 정답을 대신 말해주기보다 아이 발화를 확장(expansion)하거나 고쳐 말해주기(recast)로 자연스럽게 교정한다. 이 원리를 AI 튜터 대화 설계에 그대로 적용한다.
7. **맥락 기반 학습 (Contextual/Situated Learning)**: 아이가 실제로 마주친 상황(책, 사물, 표지판)에서 나온 텍스트를 학습 재료로 쓰면 동기와 기억 인출 단서(retrieval cue)가 강화된다.

## 6. 핵심 기능 (MVP)

### 6.1 사진 캡처 & OCR 학습 아이템 추출
**적용 원리**: 맥락 기반 학습(6-7)

- 아이가 책, 워크북, 카드, 표지판 등 영어 텍스트를 카메라로 촬영
- iOS **Vision Framework**의 텍스트 인식(`VNRecognizeTextRequest`)으로 텍스트 추출
- 추출된 텍스트에서 학습 후보 단위(단어/짧은 구/문장)를 분리
  - 문장 전체, 핵심 단어, 핵심 표현(구) 단위로 후보를 나눈다
  - 아이의 기존 학습 이력과 비교해 **새로운 항목만** 우선 후보로 제시 (이미 아는 단어는 자동 제외/저비중 처리)
- 아이가 후보 중 학습할 항목을 직접 선택 (또는 전체 자동 등록 — 아이 연령/숙련도에 따라 UX 검증 필요, 7.1 오픈 이슈 참고)
- 각 학습 아이템은 다음 메타데이터와 함께 저장됨: 원본 이미지(썸네일), 캡처 문맥(주변 문장), 캡처 일시, 발음(TTS), 간단한 뜻/사용 예

**성공 기준**: 일반적인 조명/각도의 아동 도서·워크북 사진에서 텍스트 인식 정확도, 후보 분리 품질(불용어만 남는 등 오작동 최소화)

### 6.2 초기 학습 세션 (Encoding)
**적용 원리**: 생성 효과(3), 이해 가능한 입력(5)

- 캡처 직후 짧은 학습 세션 제공: 발음 듣기(TTS) → 아이가 따라 말하기(음성 인식으로 시도 여부만 확인, 정밀 발음 평가는 하지 않음) → 뜻/예문 확인
- 이 시점에는 정답을 알려주는 데 집중 (첫 노출은 인지 부하를 낮게 유지)

### 6.3 리마인드 엔진 (간격 반복 스케줄러)
**적용 원리**: 간격 반복(1), 시험 효과(2), 바람직한 어려움(4)

- 각 학습 아이템은 개별 반복 스케줄을 가짐 (Leitner 시스템 또는 SM-2 계열 알고리즘 기반)
  - 기본 간격 예시: 1일 → 3일 → 7일 → 14일 → 30일 (정답 여부에 따라 간격 증가/축소)
  - 오답/머뭇거림이 반복되면 간격을 줄여 재노출
- 예정된 아이템은 푸시 알림으로 "그때 배운 거 기억나?" 형태로 리마인드
- 리마인드 퀴즈는 **정답을 다시 보여주는 방식이 아니라 회상을 요구하는 방식**으로 설계:
  - 원본 캡처 사진(문맥 단서)을 보여주고 빈칸 채우기
  - 뜻만 보여주고 영어로 말해보기 (음성 입력)
  - AI 튜터와의 짧은 대화 속에 자연스럽게 삽입 (6.4와 연동)
- 여러 번 연속 정답 시 "정착 완료"로 표시, 반복 빈도를 낮춤

### 6.4 AI 회화 튜터
**적용 원리**: 아동지향어/스캐폴딩(6), 생성 효과(3)

가장 차별화된 기능이자 리스크가 큰 기능. 별도 섹션(7장)에서 상세 설계.

- 아이가 최근 학습한 단어/표현을 소재로 AI와 짧은 대화 진행
- 대화는 텍스트+음성(TTS/STT) 모두 지원 (아이가 아직 타이핑에 서툴 수 있음을 고려, 음성 우선)
- 대화 중 리마인드 대상 항목이 자연스럽게 등장하도록 유도 (6.3과 연동)

### 6.5 학습 홈 / 아이용 진행 화면
- 오늘의 리마인드 목록, 최근 캡처 목록, 간단한 진행 시각화(예: 정착된 단어 수)
- 복잡한 통계/그래프보다는 아이가 이해할 수 있는 단순한 형태(스티커, 배지 등 경량 동기부여 요소)로 구성

## 7. AI 튜터 대화 설계 원칙 (상세)

기존 영어 회화 AI와의 결정적 차이. 아래 원칙을 프롬프트/시스템 설계에 명시적으로 반영해야 한다.

1. **기다림 (Wait Time)**: 아이가 응답하지 않거나 머뭇거려도 일정 시간(예: 수 초~십수 초, 침묵 감지 기반) 동안 먼저 말하지 않는다. 성인 대상 AI 튜터의 "빠른 응답성"을 이 맥락에서는 의도적으로 배제한다.
2. **대신 말해주지 않기**: 아이가 문장을 끝맺지 못해도 AI가 문장을 완성해주지 않는다. 대신 짧은 힌트나 단순화된 재질문으로 유도한다. (예: 아이가 "I want..." 하고 멈추면 AI는 "I want... an apple? a book?"처럼 선택지를 주거나, "Take your time!" 같은 격려만 하고 기다린다.)
3. **확장(Expansion)과 고쳐 말하기(Recast)**: 아이가 문법적으로 틀리거나 단순한 문장을 말해도 직접 "틀렸어"라고 지적하지 않고, AI가 올바른/조금 더 확장된 문장으로 자연스럽게 되받아 말해준다. (예: 아이 "I go school." → AI "Oh, you go to school! What do you do at school?")
4. **난이도 조절 (i+1)**: AI가 사용하는 문장의 복잡도는 아이의 최근 발화 수준보다 한 단계 높은 정도로 유지한다.
5. **긍정적 강화 우선**: 정오 판정보다 시도 자체에 대한 격려를 우선한다.
6. **학습 이력 연동**: 아이가 최근 캡처/리마인드한 단어·표현을 대화 소재로 우선 사용해 맥락 기반 재노출을 유도한다.
7. **안전장치**: 아동 대상 서비스이므로 부적절한 주제 회피, 개인정보(실명, 주소 등) 수집 회피, 대화 주제를 학습 목적 범위로 한정하는 가드레일 필요.

## 8. 대표 사용자 흐름 (User Flows)

### 8.1 캡처 → 학습 흐름
1. 앱 실행 → 카메라 버튼 탭
2. 영어 텍스트 촬영
3. Vision OCR로 텍스트 추출 → 학습 후보 단어/표현 리스트 표시
4. 아이가 학습할 항목 선택(또는 자동 전체 등록)
5. 각 항목에 대해 발음/뜻 확인하는 짧은 학습 세션 진행
6. 완료 시 리마인드 스케줄 자동 등록 (1일 후부터 시작)

### 8.2 리마인드 흐름
1. 예정일에 푸시 알림 발송 ("전에 배운 'apple' 기억나?")
2. 알림 탭 → 회상형 퀴즈 화면 진입 (원본 사진 문맥 제공)
3. 아이가 답 시도 (음성/선택)
4. 정답 여부에 따라 다음 간격 조정
5. 정답 시 짧은 긍정 피드백, 오답/모름 시 정답 노출 후 더 짧은 간격으로 재등록

### 8.3 AI 대화 흐름
1. 홈 화면에서 "말해보기" 진입
2. AI가 최근 학습 항목을 소재로 짧은 인사/질문 시작
3. 아이 발화 시도 → (7장 원칙에 따라) 기다림/힌트/확장 대응
4. 대화 종료 시 오늘 등장한 리마인드 대상 항목의 회상 성공 여부를 세션 결과에 반영

## 9. 비기능 요구사항

- **아동 개인정보 보호**: 개인식별정보 최소 수집, 보호자 동의 절차(앱스토어 아동 카테고리/국내 개인정보보호법 아동 특례 준수 필요 — 법률 검토 필요 항목으로 별도 확인).
- **오프라인 대응**: OCR과 초기 캡처 저장은 가능한 온디바이스(Vision)로 처리해 네트워크 없이도 캡처 가능해야 함. AI 대화 및 서버 동기화는 온라인 필요.
- **응답 지연**: AI 대화의 음성 인식~응답 지연이 아이의 집중력을 해치지 않는 수준이어야 함 (단, 6.1의 "기다림" 설계와는 별개로 시스템 자체 처리 지연은 최소화).
- **콘텐츠 안전성**: AI 응답에 대한 아동 대상 콘텐츠 필터링/가드레일 필수.
- **접근성**: 아직 읽기가 서툰 아이를 고려해 주요 UI 요소에 아이콘/음성 안내 병행.

---

## 10. 기술 방향 (참고용 개요)

- **플랫폼**: iOS 네이티브 (Swift/SwiftUI 권장), Vision Framework로 온디바이스 OCR
- **음성**: iOS 내장 STT(Speech framework)/TTS(AVSpeechSynthesizer) 우선 활용
- **AI 대화 엔진**: LLM API 연동 (대화 페르소나는 7장 원칙을 시스템 프롬프트로 명시)
- **리마인드 스케줄링**: 로컬 알고리즘(SM-2/Leitner 변형)으로 각 학습 아이템별 다음 노출일 계산, iOS 로컬/원격 푸시로 알림
- **데이터 저장**: 학습 아이템(캡처 이미지, 텍스트, 스케줄 상태)은 기기 로컬 우선 저장, 필요 시 계정 기반 서버 동기화는 이후 버전에서 고려

## 11. 오픈 이슈 / 추가 논의 필요 사항

1. **자동 등록 vs 선택 등록**: OCR로 추출된 모든 항목을 자동으로 학습 큐에 넣을지, 아이가 매번 선택하게 할지 — 아이 자기주도성 사용성 테스트 필요.
2. **아동 개인정보/법적 요건**: 국내 아동 대상 앱 개인정보 처리 요건, 앱스토어 Kids Category 요건 확인 필요.
3. **AI 대화 실패 처리**: 아이가 장시간 침묵하거나 전혀 응답하지 않을 때의 타임아웃/대체 흐름 정의 필요.
4. **리마인드 알림 피로도**: 알림이 지나치게 잦을 경우 아이/보호자의 앱 이탈 유발 가능 — 하루 최대 알림 수 등 정책 필요.
5. **오프라인 상태에서의 AI 대화 대체 경험**: 네트워크 미연결 시 캡처/리마인드는 가능하나 AI 대화는 불가 — 이 경우 UX 처리 방안.
