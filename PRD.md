# Brain English — PRD (Product Requirements Document)

**작성일**: 2026-07-01
**버전**: v0.2 (MVP, 리마인드 앱으로 재정의)
**작성 목적**: 뇌과학 기반 학습 원리(간격 반복, 능동적 회상, 사전 추측 효과 등)를 아이가 실제로 학습 중인 콘텐츠에 적용해, "이미 배운 것을 정확히 다시 떠올리게" 만드는 아동용 리마인드 앱의 제품 요구사항 정의

> **v0.1 → v0.2 주요 변경**: AI 대화(음성 튜터) 기능 제거, "학습 콘텐츠 제공 앱"에서 "캡처+리마인드 유틸리티"로 정체성 재정의, 캡처 방식 3종(카메라/앨범/타이핑)으로 확장, 리마인드 퀴즈 7종 다양화, 캡처(인코딩) 시점 뇌과학 기법 추가, 수학은 로드맵으로 이동.

## 1. 배경 및 문제 정의

기존 영어 학습 앱은 대부분 정해진 커리큘럼(코스)을 순서대로 따라가는 방식이다. 이 방식의 한계는 다음과 같다.

- **학습 후 방치**: 한 번 배운 표현을 다시 떠올리게 하는 장치가 없어 망각 곡선(Ebbinghaus forgetting curve)대로 잊혀진다.
- **아이의 실제 학습 맥락과 분리**: 앱이 미리 정해준 커리큘럼 콘텐츠만 학습하고, 아이가 실제로 학교·학원·원서읽기 앱 등에서 마주치는 영어와는 별개로 돌아간다.
- **본 서비스의 역할 정의**: 이 앱은 영어를 "가르치는" 앱이 아니다. 커리큘럼을 설계하거나 콘텐츠를 만들지 않는다. 아이가 이미 다른 경로(학교, 학원, 원서읽기 앱 등)로 배우고 있는 것을 **캡처해서, 며칠 뒤 정확히 다시 떠올리게 만드는 리마인드 유틸리티**다.

## 2. 제품 비전

> 아이가 오늘 실제로 배운 영어 표현을 사진·스크린샷·타이핑으로 캡처해두면, 뇌과학적으로 검증된 간격 반복과 능동적 회상을 통해 며칠 뒤 자연스럽게 다시 떠올리게 만든다.

**핵심 차별점**

1. **리마인드 앱, 학습 콘텐츠 앱이 아님**: 커리큘럼/콘텐츠를 만들지 않는다. 아이가 실제로 학습 중인 것(학교 교재, 원서읽기 앱 등)을 캡처하는 것이 유일한 콘텐츠 소스다.
2. **간격 반복 리마인드**: 학습 직후가 아니라 며칠 뒤, 정해진 간격으로 "그때 배운 거 기억나?"를 능동적으로 회상시킨다.
3. **캡처 시점부터 뇌과학 원리 적용**: 리마인드뿐 아니라 처음 캡처하는 순간(인코딩 단계)에도 사전 추측, 다중 감각 부호화 등의 기법을 적용해 기억 정착률을 높인다.
4. **유연한 캡처**: 종이책이든 디지털 원서읽기 앱이든, 사진·스크린샷·직접 타이핑 중 상황에 맞는 방식으로 캡처할 수 있다.

### 2.1 경쟁 앱과의 관계 (포지셔닝)

"사진 캡처 → 플래시카드 → 간격반복"이라는 메커니즘 자체는 이미 상용화되어 있다(예: Migaku, CapWords, Readwise Mastery). 이 앱의 차별점은 메커니즘의 발명이 아니라 다음 조합에 있다.

- **캡처 대상의 방향이 다름**: CapWords 등은 실생활 사물을 찍어 "새 단어를 발견"하는 도구인 반면, 이 앱은 아이가 **이미 배우고 있는 특정 문장·표현을 정확히 재소환**하는 도구다.
- **아동 발달을 고려한 캡처 권한 이관**: 초기엔 보호자가 캡처하고 점차 아이가 스스로 하도록 설계.
- **리마인드 방식의 다양성**: 표준 플래시카드 하나가 아니라 7종의 회상 방식(3장 참고)을 상황에 맞게 로테이션.
- "세계 최초 컨셉"이 아니라 "기존 카테고리를 아동 눈높이·실제 학습 맥락에 맞게 재구성한 버전"으로 정직하게 포지셔닝한다.

## 3. 타겟 사용자

**주 사용자 (아이)**
- 문법 지식은 거의 없음, 간단한 영어 단어/문장을 어느 정도 읽을 수 있음
- 스스로 앱을 조작할 수 있는 연령/숙련도
- 한국에 거주하며 한국어를 모국어로 사용
- 학교/학원 교재 또는 원서읽기 앱(디지털)으로 영어를 배우고 있음

**부 사용자 (보호자)**
- **캡처 초기 주체**: MVP 초반에는 보호자가 그날 배운 내용을 캡처해줌 → 아이가 익숙해지면 점차 스스로 캡처하도록 자연스럽게 이관
- 앱 설치/설정 담당

**시장/언어**: 한국어 UI, 한국 거주 아동 대상. 학습 콘텐츠 자체는 영어.

**배포 방식**: 앱스토어 정식 출시가 아니라 **개인/가정용 배포**를 우선 고려한다 (11장 참고). 이는 앱스토어 아동 카테고리 심사, 대규모 서비스에 요구되는 아동 보호 정책(연령 확인, 모니터링 체계 등)의 부담을 낮추기 위한 전략적 선택이다.

## 5. 학습 과학적 근거 (Learning Science Foundations)

### 5.1 리마인드(회상) 단계에 적용하는 원리

1. **간격 반복 효과 (Spaced Repetition Effect)**: 일정 간격을 두고 재노출할 때 장기 기억 정착률이 높다.
2. **능동적 회상/시험 효과 (Retrieval Practice)**: 스스로 떠올리려는 시도가 기억을 강화한다.
3. **생성 효과 (Generation Effect)**: 수동적 인식보다 스스로 산출(쓰기/조합)할 때 기억이 강해진다.
4. **바람직한 어려움 (Desirable Difficulty)**: 적절히 힘든 회상이 효과적이다.
5. **인터리빙 (Interleaving)**: 같은 캡처 배치나 같은 퀴즈 유형만 몰아서 반복하지 않고, 여러 날짜·유형을 섞어서 제시하면 장기 기억에 더 효과적이다.

### 5.2 캡처(인코딩) 단계에 적용하는 원리 — v0.2 신규

리마인드뿐 아니라 **처음 캡처하는 순간**에도 아래 기법을 적용한다.

1. **사전 추측 효과 (Pre-testing Effect)**: 캡처 직후 뜻을 바로 보여주지 않고, "이 뜻이 뭘 것 같아?" 짧은 객관식으로 먼저 추측하게 한 뒤 정답 공개. 틀려도 상관없이, 먼저 인출을 시도하는 것 자체가 이후 기억에 도움이 된다.
2. **즉시 미니 재노출 (Expanding Retrieval)**: 캡처 세션이 끝나기 직전, 방금 캡처한 항목들로 짧은 "0일차" 복습을 한 번 더 진행해 초기 인코딩을 강화한다.
3. **다중 감각 부호화 (Dual Coding)**: 캡처 순간 텍스트 + 원본 이미지(있는 경우) + 발음(TTS)을 동시에 제공해 여러 감각 경로로 정보를 입력한다.

### 5.3 맥락 기반 학습 (Contextual Learning)

아이가 실제로 마주친 상황(교재, 원서읽기 앱)에서 나온 텍스트를 그대로 재노출하면 동기와 기억 인출 단서(retrieval cue)가 강화된다. AI로 새로 그림을 그리거나 예문을 창작하지 않고, **원본 캡처 내용 자체를 회상 단서로 재사용**하는 것이 핵심 설계 원칙이다.

### 5.4 v0.1에서 제거된 원리 (참고)

- 아동지향어/스캐폴딩 원리(모성어 대화 방식)는 AI 대화 튜터 기능 제거와 함께 v0.2에서 제외했다. 관련 리스크(아동 음성 데이터 처리, 페르소나 튜닝 난이도, API 정책 준수 부담)가 기대 효과 대비 크다고 판단.

## 6. 핵심 기능 (MVP)

### 6.1 캡처 — 3가지 방식

**적용 원리**: 맥락 기반 학습(5.3)

캡처 대상의 소스가 물리적 교재냐 디지털 앱이냐에 따라 최적 방식이 다르므로, 세 가지를 모두 지원한다.

| 방식 | 적합한 상황 | 처리 |
|---|---|---|
| **카메라 촬영** | 종이책, 워크북, 문제집 | iOS Vision Framework OCR (`VNRecognizeTextRequest`) |
| **앨범에서 가져오기** | 원서읽기 앱 등 디지털 콘텐츠 (스크린샷) | 동일한 Vision OCR 파이프라인 적용, 화면 재촬영보다 인식률 높음 |
| **직접 타이핑** | 스크린샷이 불가능하거나, 짧은 단어라 타이핑이 더 빠를 때 | OCR 불필요, 텍스트 필드 입력 |

세 방식은 우열 관계가 아니라 **동등한 캡처 경로**다. OCR은 "있으면 편리한 기능"이며, 타이핑만으로도 앱의 핵심 루프(캡처→리마인드)는 완전히 동작한다.

**캡처 UX ("Fancy" 추출 이펙트)**
- 카메라를 비추는 동안 실시간으로 텍스트 영역을 감지해 하이라이트 표시 (Apple Live Text와 유사한 방식)
- 아이가 원하는 문장을 탭하면 카드로 "추출"되는 애니메이션 제공

**캡처 권한 이관**
- MVP 초기: 보호자가 그날 배운 내용을 캡처
- 아이가 익숙해지면: 아이 스스로 캡처하도록 UI/권한 자연스럽게 이관 (강제 전환 로직 불필요, 자연스러운 습관 형성 유도)

**캡처량 제한**: 하루 캡처량은 그날 실제 학습량 기준, 문장 단위로 20개 이내를 권장 상한으로 둔다 (리마인드 큐 과부하 방지).

### 6.2 캡처 시점 인코딩 강화 세션

**적용 원리**: 사전 추측 효과, 즉시 미니 재노출, 다중 감각 부호화 (5.2)

1. 캡처 직후, 뜻을 바로 보여주지 않고 짧은 객관식으로 뜻을 먼저 추측하게 함 → 정답 공개
2. 원본 이미지(있는 경우) + 텍스트 + 발음(TTS)을 함께 제공
3. 캡처 세션 종료 직전, 방금 캡처한 항목들로 초간단 "0일차" 미니 복습 1회 진행
4. 완료 시 정식 리마인드 스케줄 등록 (1일 후부터 시작)

### 6.3 리마인드 엔진 (간격 반복 스케줄러)

**적용 원리**: 간격 반복, 시험 효과, 바람직한 어려움, 인터리빙 (5.1)

- 각 학습 아이템은 개별 반복 스케줄 보유 (SM-2/Leitner 계열 알고리즘)
  - 기본 간격 예시: 1일 → 3일 → 7일 → 14일 → 30일 (정답 여부에 따라 조정)
- 리마인드 세션 구성 시 같은 캡처 배치나 같은 퀴즈 유형을 몰아주지 않고 섞어서 제시 (인터리빙)
- 여러 번 연속 정답 시 "정착 완료"로 표시, 반복 빈도 낮춤

**알림 정책**
- 하루 최대 알림 개수 제한 (예: 3~5개)
- 제한을 넘겨 밀린 항목은 알림 없이 **앱 내 "밀린 리마인드" 백로그**로 이동 — 데이터는 삭제되지 않고, 아이가 앱을 열면 언제든 확인 가능. 무한정 알림을 재전송하거나 강제로 항목을 폐기하지 않는다.

### 6.4 리마인드 퀴즈 — 7종

캡처한 문맥을 그대로 재사용하며, 아래 유형을 인터리빙으로 로테이션한다. AI 호출 여부를 명시해 리스크를 관리한다.

| # | 유형 | 방식 | AI 필요 |
|---|---|---|---|
| 1 | 문장 재배열 | 캡처한 문장의 단어를 뒤섞어 제시, 순서 맞추기 | 불필요 |
| 2 | 원본 빈칸 채우기 | 캡처했던 원본 텍스트(이미지 포함 가능)를 다시 보여주되 목표 단어만 가림 | 불필요 |
| 3 | 스펠링 재구성 | 단어 철자를 뒤섞고 순서 맞추기 | 불필요 |
| 4 | 단어→뜻 객관식 | 오답 선택지는 다른 캡처 단어 중에서 자동 생성 | 불필요 |
| 5 | 뜻→단어 회상 | 한글 뜻을 보여주고 영어로 타이핑/말하기, 근사 매칭으로 확인 | 불필요 (음성 시 온디바이스 STT만) |
| 6 | **듣기 딕테이션** | TTS로 문장/단어 재생 → 아이가 들은 단어를 타이핑 → 문자열 비교 | 불필요 (TTS 엔진만) |
| 7 | **주어·명사 치환 연습** | 캡처 문장의 문형(패턴)을 유지하며 주어/명사만 바꾼 변형 문장으로 연습 | **필요** (아래 6.5 참고) |

(선택, 고급 단계용) **새 문장 만들어보기**: 단어를 활용해 스스로 새 문장을 만들어보고 AI가 단발성으로 판정 — 정착 완료 단계 이후에만 노출해 AI 사용량을 자연스럽게 제한.

### 6.5 주어·명사 치환 연습 — 상세 설계

가장 AI 의존도가 높은 기능이므로 리스크를 좁게 통제한다.

**생성 방식**
- 실시간 대화가 아니라, **이벤트 기반 단발성 생성**: 캡처 시 1회 호출로 3~6개 변형 문장을 배치 생성해 저장
- 아이가 저장된 변형 세트를 모두 소진하면 그때 다시 1회 호출해 새 배치 생성 (고정된 세트를 계속 재사용해 패턴이 아니라 특정 문장을 외워버리는 것을 방지)

**치환 단어 소스 (우선순위)**
1. **아이의 캡처 단어장** — 이미 학습 중인 단어를 우선 사용 (문법 패턴 연습과 어휘 재노출을 동시에 달성, 난이도도 이미 검증된 상태)
2. **내장 정적 데이터 (부족 시 폴백)** — Dolch Sight Words / Fry's Instant Words (미국 초등 읽기 교육의 표준 학년별 단어 목록, 퍼블릭 도메인, 앱에 오프라인 데이터로 내장, API 호출 불필요)
- AI는 위 목록 안에서만 단어를 골라 문장에 끼워 넣는 역할로 제한 → 임의로 새 단어를 만들어내지 않도록 통제해 결과물의 안정성을 높임

### 6.6 홈 / 진행 화면

- 오늘의 리마인드 목록, 밀린 리마인드 백로그, 최근 캡처 목록
- 아이가 이해할 수 있는 단순한 진행 시각화(정착된 단어 수 등)

## 7. 음성(TTS) 기술 방향

- **1단계 (기본, 무료)**: iOS `AVSpeechSynthesizer`의 Enhanced/Premium 신경망 음성. iOS 설정에서 다운로드, 온디바이스·오프라인 동작, 100ms 이내 재생 시작. 최근 iOS는 이 음성이 신경망 기반으로 충분히 자연스러움.
- **2단계 (필요시, 유료)**: 1단계 품질이 부족하다고 판단되면 ElevenLabs 등 신경망 TTS API로 업그레이드 검토. 단어/짧은 문장 재생 용도라 사용량 기준 비용 부담은 크지 않음.
- 듣기 딕테이션(6.4 #6)을 포함한 모든 TTS 재생 기능은 AI/LLM 판단이 필요 없는 순수 음성 합성 기능이다.

## 8. 비기능 요구사항

- **아동 개인정보 보호**: 개인식별정보 최소 수집. 개인/가정용 배포(11장)로 시작하는 경우 상업적 아동 대상 서비스에 요구되는 연령 확인·모니터링 체계 부담은 낮아지나, 각 API 제공사의 이용약관(미성년자 대상 API 사용 조건) 확인은 별도로 필요.
- **오프라인 대응**: 캡처(OCR), 리마인드 퀴즈 1~6번, TTS 1단계는 온디바이스로 동작. 주어·명사 치환 생성(6.5)만 네트워크 필요.
- **콘텐츠 안전성**: 치환 단어 소스를 검증된 목록(캡처 단어장, Dolch/Fry)으로 제한해 AI가 부적절한 단어를 생성할 여지를 최소화.
- **접근성**: 아직 읽기가 서툰 아이를 고려해 주요 UI 요소에 아이콘/음성 안내 병행.

## 9. 기술 방향 (참고용 개요)

- **플랫폼**: iOS 네이티브 (Swift/SwiftUI), Vision Framework(OCR), Speech Framework(STT, 시도 확인용), AVSpeechSynthesizer(TTS)
- **AI 사용 범위**: 주어·명사 치환 문장 생성(이벤트 기반 단발성 호출)과 (선택) 새 문장 만들기 판정에만 한정. 실시간 대화형 AI는 사용하지 않음.
- **리마인드 스케줄링**: 로컬 알고리즘(SM-2/Leitner 변형)
- **데이터 저장**: 학습 아이템(캡처 이미지/텍스트, 스케줄 상태)은 기기 로컬 우선 저장

## 12. 로드맵 (MVP 이후)

### 12.1 수학 확장 (2순위)

- **캡처**: 원리 서술이 아니라, 캡처한 문제와 **같은 유형·다른 숫자의 새 문제**를 생성해 풀게 하는 방식 (숫자 기반 문제만, 도형 등 그림 필수 문제는 제외)
- **생성**: 단순 연산은 규칙 기반(숫자만 교체, AI 불필요), 문장제는 AI 1회 생성
- **채점**: 아이가 타이핑한 숫자 답과 정답을 코드로 비교 (AI 불필요, 100% 결정론적) — 손글씨 인식 오류나 AI의 자유서술 판정 리스크를 원천적으로 회피
- 오답 시 힌트 제공 후 재시도, 지속 오답 시 리마인드 간격 단축

### 12.2 기타

- 발음 정밀 평가(전문 API 도입 검토: Azure Pronunciation Assessment, Speechace 등)
- 보호자용 진행 리포트
- 다국어 UI, 안드로이드 확장
- 수익화 모델 검토 (현재는 완전 무료)
